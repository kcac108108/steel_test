# 강종 · 사이즈 분류 방법 설명서

---

## 1. 사전 데이터를 미리 만들어두는 이유

분류 시스템은 매번 분류를 실행할 때 전체 확정 데이터를 다시 읽으면 속도가 매우 느립니다.  
이를 해결하기 위해 두 가지 사전 데이터를 미리 구축해 놓고 빠르게 조회합니다.

### RAG 유사사례 인덱스 (ChromaDB)

- **무엇인가?**  
  과거 확정 데이터의 규격 텍스트를 AI가 이해할 수 있는 숫자 벡터로 변환하여 저장해 놓은 데이터베이스입니다.

- **왜 미리 만드나?**  
  규격 텍스트를 벡터로 변환하는 작업은 시간이 오래 걸립니다. 분류할 때마다 수십만 건을 변환하면 실용적이지 않으므로, 미리 변환해서 저장해두고 조회만 합니다.

- **언제 갱신하나?**  
  담당자가 검토·확정한 파일을 데이터 갱신 화면에서 업로드하면 자동으로 추가됩니다.  
  이미 있는 규격은 중복 추가되지 않습니다.

### 사이즈 정확매칭 사전 (size_lookup.pkl)

- **무엇인가?**  
  과거 확정 데이터 전체에서 `규격 텍스트 → 사이즈` 매핑을 딕셔너리로 구축해 놓은 파일입니다.  
  현재 100만 건 이상의 매핑 정보가 저장되어 있습니다.

- **왜 미리 만드나?**  
  분류할 때마다 확정 데이터 엑셀 파일 수십 개를 모두 읽으면 매우 느립니다.  
  미리 딕셔너리로 만들어두면 규격 텍스트로 즉시 조회가 가능합니다.

- **언제 갱신하나?**  
  담당자가 확정한 파일을 데이터 갱신 화면에서 업로드하면 새 규격→사이즈 매핑이 자동으로 추가됩니다.  
  갱신이 반복될수록 exact 매칭 건수가 늘어나 정규식·LLM에 의존하는 비율이 줄어듭니다.

## 2. 강종 분류

규격 텍스트를 입력받아 아래 순서대로 강종을 분류합니다.  
앞 단계에서 분류되면 다음 단계로 넘어가지 않습니다.

```
룰베이스 → RAG → LLM → 미분류
```

---

### 2-1. rule (룰베이스 매칭)

**어떻게 분류하나?**  
Oracle DB의 rule_base 테이블에 등록된 패턴과 규격 텍스트를 비교합니다.  
규격 텍스트가 등록된 패턴으로 **시작하면** 해당 강종으로 분류합니다.

> 예를 들어 패턴이 `JIS G3101 SS400` 이라면,  
> 규격이 `JIS G3101 SS400, SIZE 3.2X...` 처럼 이 패턴으로 **시작하는** 경우에만 매칭됩니다.  
> 규격 중간에 `JIS G3101 SS400`이 있어도 매칭되지 않습니다.

**장점**
- 가장 빠르고 정확도가 높습니다 (신뢰도 100%)
- Oracle DB에 패턴이 축적될수록 커버 범위가 넓어집니다

**단점**
- 패턴이 등록되어 있어야만 분류 가능합니다
- 새로운 형태의 규격은 룰베이스에 없으면 다음 단계로 넘어갑니다

**예시**

| 규격 텍스트 | 등록 패턴 | 분류 강종 |
|---|---|---|
| `JIS G3101 SS400, SIZE 3.2X1219X2438MM` | `JIS G3101 SS400` | JIS G3101 SS400 |
| `JIS G3131 SPHC 1.6X1000X2000MM` | `JIS G3131 SPHC` | JIS G3131 SPHC |
| `MODEL: HPE-2 SIZE 230X400X1250MM` | `MODEL: HPE-2` | HPE-2 |

**패턴은 어떻게 늘어나나?**  
담당자가 확정한 파일을 갱신하면 일관성 95% 이상인 패턴이 자동으로 Oracle DB에 추가됩니다.

---

### 2-2. rag (유사사례 검색)

**어떻게 분류하나?**  
미리 구축해둔 RAG 인덱스에서 입력 규격과 가장 유사한 과거 사례를 찾아 강종을 결정합니다.  
유사도가 **0.90 이상**인 경우에만 결과로 사용합니다.

**장점**
- 룰베이스에 패턴이 없어도 분류 가능합니다
- 규격 표현이 조금 달라도 유사한 사례를 찾아냅니다
- 과거 확정 데이터가 많을수록 정확도가 올라갑니다

**단점**
- 규격 텍스트 전체를 비교하므로 사이즈 부분이 많이 다르면 유사도가 낮게 나올 수 있습니다
- 유사도 기준(0.90) 미만이면 분류하지 않고 LLM으로 넘어갑니다

**예시**

| 입력 규격 | 유사 확정 사례 (유사도) | 분류 강종 |
|---|---|---|
| `LOT NO.2505205 TYPE SS400 SIZE 230X400` | `LOT NO.2505001 TYPE SS400 SIZE 150X300` (0.95) | SS400 |
| `COLD ROLLED COIL JIS G3141 SPCC T:0.8` | `COLD ROLLED COIL JIS G3141 SPCC T:1.0` (0.92) | JIS G3141 SPCC |

---

### 2-3. llm (AI 판단)

**어떻게 분류하나?**  
룰베이스·RAG 모두 실패한 경우 GPT(AI)에게 규격 텍스트를 보내 강종 판단을 요청합니다.

**장점**
- 어떤 형태의 규격도 시도해볼 수 있습니다
- 문맥을 이해해서 판단합니다

**단점**
- 세 가지 방법 중 정확도가 상대적으로 낮습니다
- 외부 API 비용이 발생합니다
- 인터넷이 연결되어 있어야 합니다

**예시**

| 입력 규격 | 분류 강종 |
|---|---|
| `STAINLESS WIRE ROPE 7X19 SUS304 DIA 8MM` | SUS304 |
| `SEAMLESS PIPE ASTM A106 GR.B SCH40 4IN` | ASTM A106 GR.B |

---

### 2-4. unclassified (미분류)

룰베이스, RAG, LLM 세 가지 모두 강종 분류에 실패한 경우입니다.  
결과 파일의 `시스템_강종` 컬럼이 비어있습니다.

---

## 3. 사이즈 분류

규격 텍스트를 입력받아 아래 순서대로 사이즈를 추출합니다.

```
정확매칭(exact) → 정규식(regex) → LLM → 미분류
```

사이즈 형식은 숫자를 **X**로 연결합니다. (예: `3.2X1219X2438`)

---

### 3-1. exact (정확 매칭)

**어떻게 분류하나?**  
미리 구축해둔 사이즈 사전(size_lookup.pkl)에서 동일한 규격 텍스트를 찾아  
담당자가 확정한 사이즈를 그대로 가져옵니다.

**장점**
- 가장 빠르고 정확합니다
- 담당자가 확정한 값을 그대로 사용하므로 신뢰도가 높습니다

**단점**
- 사전에 동일한 규격이 있어야만 가능합니다
- 처음 보는 규격은 다음 단계로 넘어갑니다

**예시**

| 입력 규격 | 사전의 사이즈 값 | 추출 사이즈 |
|---|---|---|
| `JIS SPHC 1.6X1000X2000MM` | `1.6X1000X2000` | 1.6X1000X2000 |
| `COLD ROLLED 0.8X1219X2438MM` | `0.8X1219X2438` | 0.8X1219X2438 |

---

### 3-2. regex (정규식 추출)

**어떻게 분류하나?**  
규격 텍스트에서 숫자, 단위(MM/IN 등), 구분자(X/*) 패턴을 분석하여 사이즈를 추출합니다.  
다양한 표기 방식을 인식하도록 규칙이 미리 정의되어 있습니다.

**장점**
- 처음 보는 규격도 숫자+단위 패턴만 있으면 추출 가능합니다
- 다양한 단위 표기를 통일된 형식으로 변환합니다

**단위 변환 규칙 예시**
- `M` → `×1000` (2M → 2000)
- `CM` → `×10`
- `"` (인치) → `IN`
- `×` (곱셈 기호) → `X`
- `COIL` → 끝에 `C` 표기

**예시**

| 입력 규격 | 추출 사이즈 |
|---|---|
| `STEEL PLATE 12.5T X 1524W X 3000L MM` | 12.5X1524X3000 |
| `PIPE OD 60.3MM X WT 3.2MM X L 6000MM` | 60.3X3.2X6000 |
| `SHEET 0.5×1000×2000MM` | 0.5X1000X2000 |
| `COIL 1.2MM X 1000MM` | 1.2X1000C |
| `BAR DIA 25MM L 3M` | 25X3000 |

---

### 3-3. llm (AI 추출)

**어떻게 분류하나?**  
정확매칭·정규식 모두 실패한 경우 GPT(AI)에게 규격 텍스트를 보내 사이즈 추출을 요청합니다.

**장점**
- 특수하거나 불규칙한 표기도 처리 가능합니다

**단점**
- 외부 API 비용이 발생합니다

**예시**

| 입력 규격 | 추출 사이즈 |
|---|---|
| `ROUND BAR 2INCH X 6FEET` | 2INX1828.8 |
| `FLAT BAR 50MM WIDTH 200MM LENGTH 3MTR` | 50X200X3000 |

---

### 3-4. unclassified (미분류)

정확매칭, 정규식, LLM 세 가지 모두 사이즈 추출에 실패한 경우입니다.  
결과 파일의 `시스템_사이즈` 컬럼이 비어있습니다.

---

## 4. 결과 파일 컬럼 설명

| 컬럼명 | 설명 |
|---|---|
| `시스템_강종` | 시스템이 분류한 강종 (미분류 시 빈 값) |
| `강종_분류방법` | 강종 분류에 사용된 방법 (`rule` / `rag` / `llm` / `unclassified`) |
| `시스템_사이즈` | 시스템이 추출한 사이즈 (미분류 시 빈 값) |
| `사이즈_분류방법` | 사이즈 추출에 사용된 방법 (`exact` / `regex` / `llm` / `unclassified`) |

---

## 5. 정확도 및 커버리지

검증·통계 화면에서 결과 파일을 업로드하면 아래 지표를 확인할 수 있습니다.

| 지표 | 계산 기준 |
|---|---|
| **커버리지** | 시스템이 분류한 건수 ÷ 전체 건수 |
| **정확도** | 시스템값과 담당자값이 일치한 건수 ÷ 둘 다 값이 있는 건수 |

> 정확도 계산은 시스템 분류값과 담당자 입력값이 **모두 있는 건**만을 대상으로 합니다.
